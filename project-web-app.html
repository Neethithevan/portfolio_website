Deploying a Static Website on AWS

Learn how to set up a secure, scalable, and cost-effective static website using AWS and Terraform.

Project Overview

This project focuses on deploying a static website on AWS using Terraform for infrastructure as code, ensuring automation and reproducibility.

Amazon S3: Securely hosts static website files.
CloudFront: Speeds up content delivery globally.
Route 53: Handles domain name resolution and management.
Certificate Manager: Ensures HTTPS security.
Terraform: Automates infrastructure deployment.

Architecture

The serverless architecture leverages AWS services to deliver a high-availability, low-latency website.

Deployment Steps

Step 1: Setup Terraform

Write Terraform scripts to define AWS resources including S3, CloudFront, and Route 53.

Step 2: Deploy Infrastructure

Use Terraform to create AWS resources in a repeatable manner.

Step 3: Configure CloudFront

Set up CloudFront distribution to enhance website speed and security.

Step 4: Secure Website with SSL

Use AWS Certificate Manager to apply SSL/TLS encryption.

Step 5: Automate Deployments

Implement CI/CD with GitHub Actions to update content seamlessly.
